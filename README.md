# Mood-to-Playlist 🎵

> Type a mood or situation — get a Spotify-style playlist generated by AI.

You describe a feeling or moment in plain words and the app instantly returns a hand-curated-feeling playlist complete with song titles, artist names, and a short "why this track fits" vibe blurb for every song. No Spotify account, no sign-up, no fuss.

## ✨ Features

- **Natural-language mood input** — anything from `"3am can't sleep"` to `"post-breakup rage cleaning"`
- **AI-curated playlists** — GPT-4o-mini selects 10–15 real songs across genres and eras
- **Per-track vibe descriptions** — each track includes a sentence on why it fits your mood
- **Creative playlist naming** — the AI generates a poetic name and overall vibe summary
- **Redis caching** — identical mood queries are served instantly from cache (X-Cache: HIT/MISS header)
- **Resilient AI calls** — automatic retry with exponential back-off on transient OpenAI errors
- **Input validation** — mood string capped at 200 characters with clear error messages
- **Fast, typed UI** — React 19 + TypeScript + Tailwind CSS, no page reloads

## 🏗️ Architecture

```
Browser (React/Vite :5173)
        │  POST /api/generate  { "mood": "..." }
        ▼
FastAPI backend (:8000)
        │
        ├─► Redis cache (:6379)   ← cache HIT → return immediately
        │         │ cache MISS
        └─► OpenAI GPT-4o-mini API
                  │
                  └─► validated PlaylistResponse → store in Redis → return to client
```

**Request flow:**
1. User types a mood in the React UI and hits **Generate**.
2. Vite's dev-server proxy forwards `POST /api/generate` to FastAPI on port 8000.
3. FastAPI checks Redis for a cached response keyed on the normalised mood string.
4. On a cache miss, FastAPI calls OpenAI, parses + validates the JSON playlist, stores it in Redis (TTL: 1 hour), and returns it.
5. The frontend renders the playlist name, description, and track cards.

## 🚀 Setup & Running

### 1 — Clone the repo

```bash
git clone https://github.com/your-org/mood-to-playlist.git
cd mood-to-playlist
```

### 2 — Configure environment variables

```bash
cp backend/.env.example backend/.env   # then edit the file
```

Open `backend/.env` and fill in your values:

```env
# Required — your OpenAI API key (https://platform.openai.com/api-keys)
OPENAI_API_KEY=sk-...

# Redis connection URL
# Local default:
REDIS_URL=redis://localhost:6379
# Redis Cloud example:
# REDIS_URL=redis://default:password@host:port
```

> **Note:** Redis is optional for development — the app degrades gracefully (no caching) when Redis is unavailable, logging a warning instead of crashing.

### 3 — Start Redis (local)

```bash
# macOS (Homebrew)
brew services start redis

# Ubuntu / Debian
sudo systemctl start redis-server

# Docker (one-liner, no install needed)
docker run -d -p 6379:6379 redis:7-alpine
```

### 4 — Start the backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Backend is now running at **http://localhost:8000**.

### 5 — Start the frontend

Open a new terminal:

```bash
cd frontend
npm install
npm run dev
```

Frontend is now running at **http://localhost:5173**.

### 6 — Verify everything works

| Check | URL | Expected |
|-------|-----|----------|
| Frontend UI | http://localhost:5173 | Mood input form |
| Backend health | http://localhost:8000/api/health | `{"status":"ok"}` |
| Vite proxy | http://localhost:5173/api/health | `{"status":"ok"}` |

## 🌍 Environment Variables

All backend configuration lives in `backend/.env` (loaded by pydantic-settings).

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `OPENAI_API_KEY` | **Yes** | — | OpenAI API key. Without this, playlist generation returns HTTP 500. |
| `REDIS_URL` | No | `redis://localhost:6379` | Redis connection URL. App runs without Redis (caching disabled). |

## 📡 API Reference

### `GET /api/health`

Returns server health status.

**Response `200`**
```json
{ "status": "ok" }
```

---

### `POST /api/generate`

Generate a playlist from a mood description.

**Request body**
```json
{ "mood": "driving through rain at night" }
```

| Field | Type | Constraints |
|-------|------|-------------|
| `mood` | string | 1–200 characters |

**Response `200`**
```json
{
  "mood": "driving through rain at night",
  "playlist_name": "Neon Puddles",
  "description": "A cinematic drift through wet streets and glowing tail-lights...",
  "tracks": [
    {
      "title": "Motion Picture Soundtrack",
      "artist": "Radiohead",
      "vibe": "Haunting and slow, mirrors the reflective solitude of a late-night drive."
    }
  ],
  "cached": false
}
```

| Field | Type | Description |
|-------|------|-------------|
| `mood` | string | Echo of the original input |
| `playlist_name` | string | AI-generated creative playlist title |
| `description` | string | 2–3 sentence overall vibe summary |
| `tracks` | array | 10–15 track objects |
| `tracks[].title` | string | Song title |
| `tracks[].artist` | string | Artist / band name |
| `tracks[].vibe` | string | Why this track fits the mood |
| `cached` | boolean | `true` if served from Redis cache |

**Response headers**

| Header | Values | Meaning |
|--------|--------|---------|
| `X-Cache` | `HIT` / `MISS` | Whether the response came from Redis cache |

**Error responses**

| Status | Condition |
|--------|-----------|
| `422` | Mood string is empty or exceeds 200 characters |
| `500` | OpenAI API error or malformed AI response (after retries) |
